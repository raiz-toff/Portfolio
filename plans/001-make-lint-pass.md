### Step 1: Ignore generated `.source/**` in ESLint

In `eslint.config.mjs`, add `".source/**"` to the `globalIgnores` array:

```js
  globalIgnores([
    // Default ignores of eslint-config-next:
    ".next/**",
    "out/**",
    "build/**",
    "next-env.d.ts",
    // Generated by fumadocs-mdx (postinstall) — never lint:
    ".source/**",
  ]),
```

**Verify**: `npm run lint 2>&1 | grep -c "\.source/"` → `0`

### Step 2: Replace the `any`s in `source.config.ts` with structural types

Add these local types near the top of `source.config.ts` (after the imports):

```ts
// Minimal structural view of the mdast / mdx-jsx nodes these plugins touch.
type MdxJsxAttribute = {
  type: string;
  name: string;
  value?: unknown;
};

type Node = {
  type?: string;
  lang?: string | null;
  value?: string;
  url?: string;
  name?: string | null;
  attributes?: MdxJsxAttribute[];
  children?: Node[];
};

type EstreeProgram = {
  type: 'Program';
  sourceType: 'module';
  body: unknown[];
};
```

Then replace every `any` annotation:

- `objectToEstree(obj: Record<string, string>): any` → `: EstreeProgram`
- In `remarkMermaid`: `(node: any, parent: any, index: number | null)` →
  `(node: Node | null, parent: Node | null, index: number | null)`.
  The body guard must also narrow `parent.children` for TypeScript — change
  `if (node?.type === 'code' && node.lang === 'mermaid' && parent && index !== null)`
  to
  `if (node?.type === 'code' && node.lang === 'mermaid' && parent?.children && index !== null)`.
- `node.children.forEach((child: any, i: number) => …)` → drop the
  annotations entirely: `node.children.forEach((child, i) => …)` (they are
  inferred from `Node['children']`).
- Every `(tree: any) =>` → `(tree: Node) =>` (three occurrences: lines 70,
  128, 151).
- In `remarkFixJsxHtml` and `remarkStyleStringToObject`: `(node: any)` →
  `(node: Node | null)`. Where the code does
  `if (!node || typeof node !== 'object') return;` keep it — it still
  narrows correctly.
- In `remarkFixJsxHtml`, the line `attr.value = rewriteInternalPath(attr.value)`
  is inside `typeof attr.value === 'string'` — with `value?: unknown` this
  narrows fine; no cast needed.

Do NOT change any runtime logic — types only, plus the one guard rewrite.

**Verify**: `npm run lint 2>&1 | grep -c "source.config.ts"` → `0`
**Verify**: `npm run types:check` → exit 0

### Step 3: Lint-safe client-only values in the three placeholder components

In each file add a module-level helper and switch to `useSyncExternalStore`.
The rendered output must be identical: placeholder on the server / first
paint, real value on the client.

`src/components/greeting.tsx` — replace the `useState` + `useEffect` block
(current lines 22-30) with:

```tsx

    subscribeNever,

      : greetingForHour(hour);
```

Update the react import to `import { useSyncExternalStore } from "react";`
(remove `useEffect, useState` if now unused). Keep the file's explanatory
comment and the `suppressHydrationWarning` attribute as-is.
